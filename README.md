## Read Uncommitted
- T2 видит изменения T1 до COMMIT (грязное чтение)

## Read Committed
- T2 видит изменения T1 только после COMMIT

## Repeatable Read
- T2 блокируется, пока T1 не завершит изменения

## Serializable
- T2 блокируется полностью для любых конфликтующих операций
